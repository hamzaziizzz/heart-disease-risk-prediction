# requirements.txt
# =================
# Pinned dependencies for the Heart Disease MLOps project -- runtime +
# testing/linting only (what CI and the Docker image actually execute).
# Source of truth is Pipfile/Pipfile.lock; regenerate the runtime section with
# `pipenv requirements` and the test-tool section below by cross-referencing
# `pipenv requirements --dev` for pytest/httpx/flake8 and their direct deps.
#
# Jupyter and the rest of Pipfile's [dev-packages] notebook stack are
# DELIBERATELY EXCLUDED: neither CI nor the Docker image run notebooks, and
# including it inflated the API image from ~600MB to 2.5GB+ (it also drags in
# an unrelated 300MB nvidia-nccl-cu12 wheel). For local notebook work, use the
# pipenv virtualenv directly: `pipenv run jupyter lab`.
#
# Resolved against Python 3.14 (see `[requires]` in Pipfile), which is why CI
# and the Dockerfile also pin 3.14 -- several transitive pins here (numpy,
# scipy, xgboost, scikit-learn) declare `python_version >= '3.11'/'3.12'` and
# will not resolve on older Pythons.
#
# mlflow is pinned to 3.13.0 (not the latest 3.14.0) because 3.14.0's
# `mlflow.assistant.skill_installer` unconditionally imports
# `importlib.abc.Traversable`, which was removed in Python 3.14 -- `mlflow ui`
# crash-loops on startup otherwise.
#
# pywin32 / pywinpty are excluded: pipenv pulls them in transitively (via
# `docker`/`jupyter` resolution on this Windows dev machine) with no
# sys_platform marker, and neither has a Linux wheel.
#
# Install with:
#   python -m venv .venv && source .venv/bin/activate   (Windows: .venv\Scripts\activate)
#   pip install -r requirements.txt
# or, preferably, use pipenv directly: `pipenv install` (add --dev for jupyter/testing tools)
